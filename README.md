# aikandidattest

## Requirements

- [uv](https://docs.astral.sh/uv/) — package manager
- Python 3.13 (installed automatically by uv via `.python-version`)

## Setup

```bash
# Install dependencies (creates .venv automatically) and pre-commit hooks
make install

# Or manually:
uv sync
uv run pre-commit install
```

### Activate the virtualenv (optional)

Activating the venv lets you drop the `uv run` prefix for the rest of the session:

```bash
source .venv/bin/activate
# now you can run: pytest, cz commit, ruff check, etc. directly
deactivate  # when done
```

## Development

### Run tests

```bash
make test
# or: uv run pytest
```

Coverage report is written to `htmlcov/` after each run.

### Lint & format

```bash
make lint
# or: uv run ruff check --fix . && uv run ruff format .
```

Pre-commit runs both automatically on every commit.

### Commits

Commits must follow the [Conventional Commits](https://www.conventionalcommits.org/) specification, enforced by the `commitizen` pre-commit hook.

Use the interactive prompt to create a valid commit:

```bash
make commit
# or: uv run cz commit
```

### Version bumping

Versions are derived automatically from commit history:

| Commit type | Version bump |
|---|---|
| `fix:` | patch (0.0.x) |
| `feat:` | minor (0.x.0) |
| `feat!:` / `BREAKING CHANGE` | major (x.0.0) |

```bash
make bump       # bump version, update pyproject.toml + __init__.py, tag, update CHANGELOG.md
make bump-dry   # preview next version without applying
```
